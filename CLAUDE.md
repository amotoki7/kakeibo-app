# kakeibo-app

家計簿管理アプリ。収支の記録・集計・可視化を行う。

## Git 運用ルール

### 基本方針
- コードを変更するたびに、必ずコミットして GitHub にプッシュする
- コミットは論理的な単位でまとめる（1機能・1バグ修正 = 1コミットを目安）
- main ブランチへの直接プッシュを基本とするが、大きな変更はブランチを切る

### コミット手順
1. 変更内容を確認: `git diff` / `git status`
2. ファイルをステージング: `git add <ファイル名>`（`git add .` は避け、ファイルを明示する）
3. コミット: `git commit -m "<メッセージ>"`
4. プッシュ: `git push origin main`

### コミットメッセージ規則
- 日本語または英語、どちらでもよい
- 変更の「なぜ」を中心に書く
- 例: `収支一覧画面に月フィルターを追加`、`fix: 合計金額が負値で表示される不具合を修正`

### プッシュのタイミング
- ファイルを編集・追加・削除したとき（どんな小さな変更でも）
- 作業を中断・終了するとき
- 動作確認が取れた区切りごと
